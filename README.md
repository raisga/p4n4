> Self-hosted IoT + GenAI + Edge AI platform built on Docker Compose.

p4n4 is an open-source, multi-stack platform for building end-to-end IoT pipelines with local AI inference, composed of three Docker-based service stacks — **MING** (IoT), **GenAI**, and **Edge AI**.

---

## What can you build?

- **Home energy monitor** — collect power readings from smart plugs over MQTT, store them in InfluxDB, visualise trends in Grafana, and have Ollama summarise daily usage in plain English via n8n.
- **Predictive maintenance** — stream vibration or temperature data from industrial sensors, run an Edge Impulse anomaly-detection model on-device, and trigger automated alerts or workflows through Node-RED.
- **Local AI assistant for sensor data** — ask questions like "Why did my power spike at 2am?" or "Which sensor has been most active this week?" and get answers generated by Ollama from your own InfluxDB data — no external API calls, no data leaving your network.
- **Rapid IoT prototyping** — use the hardware reference designs in `shared/hw`, flash a board, point it at Mosquitto, and have a full observability stack running in minutes with `p4n4 up`.
- **Automated greenhouse control** — read soil moisture, temperature, and humidity from sensors, process the data through Node-RED rules, and trigger actuators (irrigation, ventilation) while logging everything for Grafana dashboards and AI-driven grow-cycle reports.

## Architecture

```
  Sensors / Devices
        │ MQTT
        ▼
  ┌─────────────────────────────────────────┐
  │  stacks/iot  (MING stack)               │
  │  Mosquitto ──► Node-RED ──► InfluxDB    │
  │                                Grafana  │
  └─────────────────────────────────────────┘
  ┌─────────────────────────────────────────┐
  │  stacks/ai  (GenAI stack)               │
  │  Ollama · Letta · n8n                   │
  └─────────────────────────────────────────┘
  ┌─────────────────────────────────────────┐
  │  stacks/edge  (Edge AI stack)           │
  │  Edge Impulse Runner                    │
  └─────────────────────────────────────────┘

  All stacks share the p4n4-net Docker bridge network.

  shared/lib ──► client/cli · client/api · client/dashboard
```

## Getting Started

### Clone with submodules

```bash
git clone --recurse-submodules https://github.com/raisga/p4n4.git
cd p4n4

# If already cloned without submodules:
git submodule update --init --recursive
```

### Quick Start (CLI)

```bash
pip install p4n4
p4n4 init my-project
cd my-project
p4n4 up
```

The CLI scaffolds compose files, generates secrets, and starts the stacks in the correct order.

### Manual Setup (without CLI)

```bash
# IoT stack
cd stacks/iot
cp .env.example .env  # edit passwords
docker network create --driver bridge --subnet 172.20.0.0/16 p4n4-net
docker compose up -d && cd ../..

# AI stack (after IoT is healthy)
cd stacks/ai
cp .env.example .env  # INFLUXDB_TOKEN must match stacks/iot/.env
docker compose up -d && cd ../..

# Edge stack (optional, requires Edge Impulse account + .eim model)
cd stacks/edge
cp .env.example .env
docker compose up -d
```

> **Cross-stack secrets:** `INFLUXDB_ADMIN_TOKEN`, `INFLUXDB_ORG`, and `INFLUXDB_BUCKET` must be identical across all stack `.env` files.
